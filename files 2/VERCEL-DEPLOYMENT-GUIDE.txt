## OPTION 1: DEPLOY FROM GITHUB (RECOMMENDED)

### STEP 1: PUSH TO GITHUB

1. Go to your GitHub repo
2. Delete ALL existing files
3. Upload ALL files from bayside-complete-site.zip
4. Make sure these files are uploaded:
   - index.html
   - package.json
   - vercel.json ← IMPORTANT!
   - vite.config.js
   - /src folder (all files)
   - /public folder (all files)
5. Commit changes

### STEP 2: CONNECT VERCEL TO GITHUB

1. Go to https://vercel.com
2. Sign in (or create account)
3. Click **"Add New..."** → **"Project"**
4. Click **"Import Git Repository"**
5. Find your GitHub repo and click **"Import"**

### STEP 3: CONFIGURE BUILD SETTINGS

Vercel should auto-detect Vite, but verify:

**Framework Preset:** Vite
**Build Command:** npm run build
**Output Directory:** dist
**Install Command:** npm install

If these aren't set, configure them manually.

### STEP 4: DEPLOY

1. Click **"Deploy"**
2. Wait 2-3 minutes for build
3. Once complete, you'll get a URL like:
   `your-site.vercel.app`
4. Test the site!

### STEP 5: ADD CUSTOM DOMAIN

1. In Vercel project settings
2. Go to **"Domains"**
3. Add: `baysidewellnessandcounseling.com`
4. Vercel will give you DNS records
5. Update Namecheap DNS (see below)

## CONFIGURING YOUR CUSTOM DOMAIN ON VERCEL

Once deployed, you need to point your domain to Vercel.

### VERCEL WILL PROVIDE DNS RECORDS

**For root domain (baysidewellnessandcounseling.com):**
- Type: A
- Value: 76.76.21.21 (Vercel's IP - might be different, check Vercel)

**For www subdomain (www.baysidewellnessandcounseling.com):**
- Type: CNAME
- Value: cname.vercel-dns.com

### UPDATE NAMECHEAP DNS

1. Log into Namecheap.com
2. Go to Domain List
3. Click **"Manage"** next to your domain
4. Click **"Advanced DNS"** tab
5. Delete old records (if any)
6. Add Vercel's records:

**Add A Record:**
- Type: A Record
- Host: @
- Value: 76.76.21.21 (or Vercel's IP)
- TTL: Automatic

**Add CNAME Record:**
- Type: CNAME Record
- Host: www
- Value: cname.vercel-dns.com
- TTL: Automatic

7. Save all changes
8. Wait 15-60 minutes for DNS propagation

## TROUBLESHOOTING COMMON ERRORS

### ERROR: "Build Failed"

**Check:**
- vercel.json is uploaded ✓
- package.json is uploaded ✓
- All files from zip are uploaded ✓

**In Vercel dashboard:**
- Go to Project Settings → Build & Development Settings
- Framework Preset: **Vite**
- Build Command: **npm run build**
- Output Directory: **dist**
- Node Version: **18.x** (or latest)

### ERROR: "404 Not Found" or Blank Page

**Hash Routing Issue:**
Your site uses hash routing (#/about), which should work fine.

If you see errors:
1. Check browser console (F12)
2. Look for errors
3. Verify all files uploaded correctly

**If pages don't load:**
- Make sure you're testing with the hash:
  - ✓ `your-site.vercel.app/#/about`
  - ✗ `your-site.vercel.app/about`

### ERROR: "Module Not Found"

**Missing Dependencies:**
1. Make sure package.json is uploaded
2. Vercel should run `npm install` automatically
3. Check build logs in Vercel dashboard

### ERROR: "SEO tags not showing"

**react-helmet-async not installed:**
1. Check Vercel build logs
2. Should see: "Installing dependencies..."
3. Should install: react, react-dom, react-helmet-async
4. If not, redeploy

## VERCEL vs NETLIFY - KEY DIFFERENCES

### SIMILARITIES:
- Both free for hobby projects
- Both support custom domains
- Both auto-deploy from GitHub
- Both provide SSL certificates

### VERCEL ADVANTAGES:
- Faster build times
- Better for Next.js (if you upgrade later)
- Great serverless functions support
- Cleaner dashboard

### NETLIFY ADVANTAGES:
- Better form handling (you're using Formspree, so not relevant)
- More generous bandwidth limits
- Easier redirect configuration

**For your site: Both work great!** Choose whichever you prefer.
